# Credits

OLO is maintained by Olo Labs and the contributors who improve its modules, workflow scenarios, runtime contracts, and documentation.

## Project stewardship

| Area | Credit |
|------|--------|
| Project direction and monorepo stewardship | Olo Labs |
| Architecture, module boundaries, and contributor documentation | Olo Labs and contributors |
| Workflow scenario library | Scenario owners and contributors |
| Runtime, worker, SPI, and core implementation modules | Module owners and contributors |

## Contributor credit policy

Add credit when someone makes a meaningful contribution to a module, scenario, public contract, or documentation set. Good credit entries name the area and the work, not just the person.

Examples:

- `Workflow scenarios: <name> for the capacity-planning preset and README examples.`
- `Runtime SPI: <name> for improving hook contracts and tests.`
- `Documentation: <name> for architecture diagrams and contributor onboarding.`

## How to update credits

1. Add the contributor or owner under the relevant area.
2. Link the pull request or issue when available.
3. Keep entries factual and concise.
4. Do not remove existing credit unless it is inaccurate and the maintainers agree.

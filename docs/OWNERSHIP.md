# Ownership and review

OLO uses module ownership to keep contributions fast, respectful, and technically grounded.

## Ownership model

- A module owner is responsible for the health of a path, not for doing all the work.
- Contributors are encouraged to improve owned areas when they keep changes scoped and request the right review.
- Review should explain tradeoffs and next steps clearly enough that new contributors can learn the system.
- Ownership and contributor credit should be visible in docs, issues, and pull requests.

## Review map

The canonical review map lives in [../OWNERS.md](../OWNERS.md). Use it when opening a pull request or deciding which README should change with a code change.

## Contributor recognition

The canonical credit file is [../CREDITS.md](../CREDITS.md). Add credit for significant module, scenario, runtime, or documentation work.

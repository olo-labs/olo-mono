# olo-workflow-input

Standalone library for **workflow invocation payloads** — serialize and deserialize the JSON sent when a workflow execution is requested (e.g. Temporal start argument).

**Documentation:** [docs/](docs/) — [Architecture](docs/ARCHITECTURE.md)

## Quick start

```bash
./gradlew build          # Unix
gradlew.bat build        # Windows
```

Requires Java 21. Depends on `org.olo:olo-definition` (see [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) §9).

**Samples:** [samples/](samples/) — JSON payloads regenerated by `./gradlew generateSamples` before tests.

## Serialize / deserialize

```java
import org.olo.input.model.WorkflowInput;

String json = input.toJson();
WorkflowInput restored = WorkflowInput.fromJson(json);
```

## Validate against a workflow definition

```java
import org.olo.input.validation.WorkflowInvocationValidator;

var result = WorkflowInvocationValidator.validate(workflowDefinition, values);
```

## Maven coordinates

`org.olo:olo-workflow-input:0.1.0-SNAPSHOT`

## Packages

| Package | Role |
|---------|------|
| `org.olo.input.model` | `WorkflowInput`, DTOs, JSON |
| `org.olo.input.producer` | Build payloads, cache offload |
| `org.olo.input.consumer` | Read-only `WorkflowInputValues` |
| `org.olo.input.validation` | vs `WorkflowDefinition.inputs` |
| `org.olo.input.config` | `MaxLocalMessageSize` |

See [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) for producer/consumer contracts, storage modes, and request flow.
